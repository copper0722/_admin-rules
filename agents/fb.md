### Popularity signal (CRITICAL — Copper directive 2026-04-20)

**`post-pattern-analysis.md` is premature — do NOT rely on it.** It was a single-case (OTC) observation, not a data-derived model. Every invocation, re-derive the pattern empirically:

```sql

       length(content) AS len
FROM posts
WHERE comments_count_api IS NOT NULL
  AND length(content) > 300
  AND topic IS NOT NULL
ORDER BY comments_count_api DESC
LIMIT 30;
```

Read those 30 posts in full. Observe: topic distribution, opening-line patterns, length distribution, use of numbered case sections vs opinion pieces, emoji density, self-reference vs 3rd-person. That is the empirical popularity model. Use it to score today's candidate.

### Style standards

- zh-TW Taiwan academic terminology (never PRC simplified / PRC-term). Law §1.1.
- Quantitative anchor MUST appear (RR/HR, n=, p, effect size, incidence). Empty platitudes rejected.
- Clinical hinge MUST appear (what does a Taiwan nephrologist do differently after reading?).

- Length target: 650-1,200 chars (matches Copper's historical popular-post length).
- No "Generated by AI" disclaimer in the draft (Copper removes those himself on publish).
- Emoji sparingly (1-3 total), Copper's idiom: 🩺 🤔 😅 — never cute/excessive.

### Source fidelity

- Cite original study/newsletter with DOI/URL + journal/publisher + date.
- If the source is a newsletter summary, pull the underlying paper first (events.db has DOI/PMID).
- OA paper → read the paper. Paywall → read abstract + Gemini grounded search, explicitly mark speculation.
- Never fabricate numbers. If a stat isn't in the source, say so in the draft (Copper can cut).

### Facebook post mechanics

- Line 1-2 = hook (tension / story / counterintuitive). Dispatch spec: "hook 要有張力".

## Hard gates (block draft if violated)

1. **Source of truth** — primary source MUST be peer-reviewed paper / gov document / independently-verified original. Digest pages (Healio/Drugs.com/Scientific American/Nature News/Science News/FDA alert abstract) are **discovery tips only**, never citation-worthy by themselves. If primary source unreachable → disqualify.
2. **First-hand data** — every quantitative claim in the draft (n, RR/HR, %, effect size) MUST come from the paper's own Results / Tables / Figures. No digest-page-paraphrased numbers. Digest version may disagree with paper; when it does, paper wins and the divergence is flagged.
3. **Causal humility** — if source design is observational (cohort, case-control, registry, MR), draft must explicitly note "相關 ≠ 因果" or spell out residual confounding. No causal language for non-RCT evidence.
4. **No nephrology technical deep-dive** — dialysis-specific mechanism essays have poor FB engagement (per empirical rubric Tier 4). Redirect to wiki agent, don't draft.
5. **No 30-day same-study repeat** (revised 2026-04-20 by Copper "recent hot topic in fb: glp1-ra"): dedup is **DOI-level, not topic-level**. Agents MUST block only when the specific paper / DOI / event URL already appeared in a post within 30 days. Same **topic** appearing repeatedly is expected behaviour for hot topics; 疲勞 signal is instead encoded in rubric scoring (hot topic +bonus, recent same-study -penalty).
   - `SELECT 1 FROM posts WHERE datetime > date('now','-30 days') AND (content LIKE '%{doi}%' OR content LIKE '%{specific_paper_marker}%')` returns → disqualify.
   - `WHERE content LIKE '%{generic_topic}%'` is NOT a disqualifier; it feeds the hot-topic-bonus logic in Phase D scoring instead.
